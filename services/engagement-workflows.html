Engagement Workflow Guide

For each engagement pattern, this document specifies the exact UI workflow — which tabs to visit, what to click, and what the client sees at each step. Green-numbered steps are client-facing interactions; blue are internal.

Business Engagements

Landscape Scan 1–2 days
Business
"What does the evidence look like?" Fast scoping of the research landscape. Ideal for proposals, feasibility checks, and initial briefings.
1
Create project from research question
Click + New Project in the top bar. Enter the client's research question. Use Dev mode for speed. The Research Planner generates a YAML config automatically.
2
Run Discovery
Go to Pipeline tab. Click the Discovery stage card and run it. This searches 10 databases and builds the paper corpus. Check paper count on completion.
3
Run pipeline through Insights
On Pipeline, click Run Pipeline and select the Landscape preset (Discovery → Abstract Enrichment → Relevance Gate → Classification). This gives corpus shape without deep extraction.
4
Review corpus overview
Open Insights tab. Capture: year distribution, source breakdown, top-cited papers, methodology distribution, gate stats (pass/review/fail). This is your landscape snapshot.
5
Client handoff: share Insights + paper count
Screenshot or export the Insights dashboard. Deliver: "We found X papers across Y databases. Here's the evidence landscape. Recommend [Evidence Build / Deep Analysis] for full synthesis."
Evidence Build 2–3 days
Business
"Build me a structured evidence base." Thorough extraction and full-text analysis. Produces the CSV evidence table the client's analysts can work with.
1
Create project and run Discovery
Same as Landscape Scan steps 1–2. Ensure corpus is built.
2
Run full pipeline (Evidence Build preset)
On Pipeline, click Run Pipeline. Select the Evidence Build preset or run stages through Results Extraction. This runs: Abstract Enrichment → Gate → Extraction → OA Retrieval → Full Text → Classification → Data Audit → Results.
3
Quality-check in Paper Browser
Open Pipeline → click individual stage cards to see per-paper results. Spot-check extraction quality on 5–10 papers. Review any gate=review papers and accept/reject manually.
4
Export evidence table
From the paper list, use the Export CSV button. This produces the 29-column evidence table. Also export BibTeX and RIS reference files.
5
Client handoff: CSV + references + platform access
Deliver: evidence table CSV, BibTeX/RIS files. Give client access to Ask tab for ad-hoc questions against the corpus.
Deep Analysis 3–5 days
Business
"Give me the full picture." Tiered analysis with credibility scoring, evidence grounding, and meta-analysis. The most comprehensive engagement.
1
Create project, run Discovery
Same as above. For Deep Analysis, consider running Discovery in Full mode for better scoring.
2
Run full pipeline with Thorough preset
On Pipeline, click Run Pipeline → select Thorough or Deep Analysis preset. This runs all 14 stages including Credibility, Evidence Grounding, Cross-Validation, and Meta-Analysis.
3
Review tiering and credibility
Check paper tiers on the paper list (T1/T2/T3). Open stage detail views for Credibility and Evidence Grounding to review per-paper scores. Manually override tiers if needed.
4
Review meta-analysis results
Open Insights for disagreement analysis, methodology distribution, and consensus findings. Use Map to visualise the evidence network — identify clusters and fault lines.
5
Generate report
Go to Reports tab. Generate with insight or executive tone. Download the self-contained HTML report with APA citations and source quotes.
6
Client handoff: report + platform access
Deliver: HTML report, CSV evidence table, BibTeX/RIS. Give client access to Ask (follow-up questions), What If (sensitivity testing), and Map (evidence visualisation).
Rapid Response 2–4 hours
Business
"I need an answer by Friday." Top 50 papers, mid-tier analysis, executive brief. For urgent decisions and board prep.
1
Create project in Dev mode
Click + New Project, enter question, use Dev mode for fastest config generation.
2
Run Discovery, then Rapid preset
Run Discovery, then on Pipeline select Rapid preset. This runs core stages only (Gate → Extraction → Results → Meta-Analysis → Report) on top 50 papers by relevance.
3
Quick QA via Ask tab
Open Ask tab. Test 2–3 key questions the client needs answered. Verify answers are grounded and citations are relevant. This takes seconds per question.
4
Generate executive brief
Go to Reports. Generate with executive or insight tone. This produces a concise, action-oriented brief.
5
Client handoff: brief + Ask access
Deliver: executive brief. Give client access to Ask tab for any follow-up questions that arise during board discussion.
Hypothesis Test 2–3 days
Business
"Is this claim actually true?" Deep extraction, credibility assessment, and evidence grounding focused on a specific claim. For due diligence.
1
Frame question as falsifiable hypothesis
Create project with a specific claim-testing question (e.g. "Does X actually improve Y?"). The config should have tight inclusion criteria focused on the claim.
2
Run full pipeline with credibility + grounding
Run Discovery, then full pipeline including Credibility and Evidence Grounding stages. These are critical for hypothesis testing — they score each paper's methodological rigour and verify claims against source text.
3
Sensitivity test the hypothesis
Open What If tab. Filter to T1 papers only — does the conclusion hold? Remove the oldest studies — still holds? Remove a specific methodology — still holds? This stress-tests the claim.
4
Check disagreement landscape
Open Map tab. Look for red disagreement nodes. Click each to see which papers are on which side and why. This reveals the nuance behind a yes/no answer.
5
Generate report framed around the claim
Go to Reports. Generate with the original hypothesis as the question. Use insight tone for a direct verdict.
6
Client handoff: verdict + sensitivity results + platform
Deliver: report with clear verdict, sensitivity analysis screenshots from What If, evidence map screenshot showing agreement/disagreement. Platform access for Ask follow-ups.
Conflict Analysis 3–5 days
Business
"Why do studies disagree about this?" Structured analysis of conflicting findings with attribution to methodology, data, or context differences.
1
Build corpus on the contested topic
Create project with the contested question. Run Discovery with broad search terms to capture both sides of the debate.
2
Run full pipeline with all quality stages
Run all 14 stages via Pipeline. Credibility and Evidence Grounding are especially important here — they determine which side of a disagreement has stronger evidence.
3
Explore disagreements on Evidence Map
Open Map tab. Red nodes are disagreements. Click each to see both sides, which papers support each position, and the attributed reason (methodology, data, context, sample size). Toggle off pattern nodes to isolate the conflict structure.
4
Decompose via What-If filters
Open What If. Filter by methodology type — do RCTs agree with observational studies? Filter by year — did newer studies resolve the conflict? Filter by tier — is it only weak studies that disagree?
5
Ask targeted questions
Open Ask tab. Ask: "Why do studies disagree on X?", "Which methodological differences explain the conflicting findings?", "What would resolve this disagreement?"
6
Client handoff: conflict report + evidence map + platform
Deliver: report focused on the disagreement, evidence map screenshots annotated with conflict lines, sensitivity analysis showing which filters resolve/preserve the conflict. Full platform access.

Academic Engagements

Meta-Analysis 2–4 days
Academic
Quantitative synthesis of findings across studies. Full extraction of results, weighted aggregation by study quality, and disagreement assessment.
1
Configure with precise research question
Create project. Then open Config tab to refine: tighten inclusion/exclusion criteria, set methodology vocabulary for the domain, and configure credibility dimensions. Set research_type: meta_analysis.
2
Run Discovery in Full mode
Run Discovery from Pipeline in Full mode (Sonnet scoring) for better relevance ranking. Aim for high recall — the gate will filter later.
3
Run complete pipeline (all 14 stages)
Run full pipeline with the Thorough preset. All stages are needed: Gate → Extraction → OA Retrieval → Full Text → Classification → Data Audit → Results → Dataset Linker → Credibility → Evidence Grounding → Cross-Validation → Meta-Analysis → Report.
4
Review cross-validation and grounding
Click the Cross-Validation stage card on Pipeline to see the cross-validation report. Check Evidence Grounding stage detail for verification rates. Flag any papers with low grounding coverage for manual review.
5
Robustness testing
Open What If. Test: exclude T3 papers — do effect sizes hold? Filter to last 5 years only — any temporal trend? Exclude single most-cited study — is the finding driven by one paper?
6
Generate academic report
Go to Reports. Generate with academic tone. This produces methodology-first structure with APA citations, appropriate hedging, and explicit limitations.
7
Deliver: report + evidence table + references + platform
Deliver: academic report (HTML), evidence table (CSV for statistical analysis), BibTeX + RIS files, verified quotes export. Platform access for Ask (sub-questions), What If (sensitivity), Map (visualising the evidence network for thesis figures).
Systematic Review 2–5 days
Academic
PRISMA-aligned comprehensive synthesis. Protocol-driven screening, structured extraction, quality assessment, and narrative synthesis.
1
Configure with PRISMA-aligned criteria
Create project. Open Config to define precise inclusion/exclusion criteria matching the review protocol. Set research_type: systematic_review. Configure all discovery sources.
2
Run Discovery across all sources
Run Discovery in Full mode. Check Insights → source distribution to verify all databases returned results. The PRISMA flow on the Pipeline tab tracks identification counts.
3
Screen with Gate + manual review
Run Abstract Enrichment → Relevance Gate. Then review gate=review papers manually: click papers on Pipeline → Gate stage detail, and accept/reject borderline cases. This is the dual-screening step.
4
Run remaining pipeline stages
Run the remaining stages: Extraction through Meta-Analysis. Use Full mode for Extraction and Results for highest-quality output on the included papers.
5
Generate report with academic tone
Go to Reports. Use academic tone. The report includes the PRISMA evidence flow, methodology section, and explicit limitations — all required for systematic review publication.
6
Deliver: report + PRISMA data + evidence table + references
Deliver: academic report, CSV for their own analysis, BibTeX/RIS, PRISMA flow data. Platform access for exploring sub-questions via Ask and testing robustness via What If.
Scoping Review 1–3 days
Academic
Map the breadth of evidence on a topic. Broader inclusion, classification-focused extraction, gap identification. Ideal for identifying research directions and grant framing.
1
Configure with broad inclusion criteria
Create project with a broad question. Open Config and set wide inclusion criteria. Set research_type: scoping_review.
2
Run Discovery + Classification pipeline
Run Discovery, then the pipeline through Classification. Deep extraction and credibility are less critical here — the goal is to map breadth, not depth.
3
Explore the landscape
Open Insights for methodology and year distributions. Open Map to see how the field clusters. Identify gaps: what methods are underrepresented? What years have sparse coverage?
4
Deliver: landscape report + evidence map + gap analysis
Generate report with academic tone. Deliver with evidence map screenshots showing field structure. Use Ask to answer "What areas are underresearched?" and "Where is the field moving?"
Rapid Evidence Assessment 4–8 hours
Academic
Streamlined synthesis for conference deadlines and grant submissions. Focused search, lighter extraction, priority analysis of top studies.
1
Create project, run Discovery in Dev mode
Speed is key. Use Dev mode throughout. Create project, run Discovery immediately.
2
Run Rapid preset pipeline
On Pipeline, select Rapid preset. This runs core stages on the top papers by relevance: Gate → Extraction → Results → Meta-Analysis.
3
Quick corpus interrogation
Open Ask tab. Ask the specific questions needed for the grant or conference abstract. Each answer comes in 3–8 seconds with citations.
4
Deliver: brief + Ask access for deadline-day questions
Generate report with insight tone (concise, direct). Deliver report + Ask access so they can query the corpus as they write their submission.
Literature Survey 1–2 days
Academic
Broad landscape mapping with narrative synthesis. For thesis introductions, background chapters, and identifying themes.
1
Create project with broad topic
Create project. Set research_type: literature_survey in Config. Wide inclusion criteria.
2
Run core stages pipeline
Run Discovery → Gate → Extraction → Classification → Results → Meta-Analysis. Skip credibility and grounding for speed.
3
Requery for each thesis section
Open Questions tab. Submit different questions for each chapter/section: "What are the theoretical frameworks for X?", "What empirical methods are used to study Y?", "What are the open questions in Z?" Each generates a focused report.
4
Deliver: section-specific reports + BibTeX + platform
Deliver: one report per thesis section, combined BibTeX file, evidence map for the introduction figure. Ask access for on-demand questions as they write.

Content Creator Engagements

Long-Form Articles & Newsletters 2–3 days
Content Creator
Research brief as outline, verified quotes as evidence, source links for every claim. The most well-sourced piece your readers have ever seen.
1
Create project from content angle
Translate the article thesis into a research question. Create project. Run Discovery to build the corpus.
2
Run pipeline through Results + Meta-Analysis
Run the pipeline on Pipeline. Include Evidence Grounding for verified quotes the creator can use verbatim.
3
Generate research brief
Go to Reports. Generate with insight tone — this produces a clear, direct brief that maps to article structure. Key findings become section headings.
4
Pull verified quotes
Export the verified quotes file. These are exact-match quotes from source papers — safe to use verbatim in the article with proper attribution.
5
Creator writes with Ask tab open
Give platform access. As they write, they use Ask to check facts ("Is it true that X?"), find specific stats ("What's the average effect size for Y?"), and explore angles ("What's the contrarian view on Z?").
YouTube Scripts 2–3 days
Content Creator
Key findings as narrative beats, evidence table for on-screen data, evidence map as an animatable visual.
1
Build corpus and run pipeline
Create project from the video's central question. Run Discovery + full pipeline including Meta-Analysis.
2
Generate insight-tone report for script beats
Go to Reports. Generate with insight tone. Each key finding becomes a narrative beat. Consensus findings become "here's what the research agrees on" segments.
3
Capture visual assets
Open Insights — screenshot charts for B-roll (year distribution, methodology breakdown). Open Map — capture the evidence network for animated explainer segments. These are screen-share ready.
4
Find the controversy angle
Open Map — click red disagreement nodes. Open Ask and ask "What's the most controversial finding?" and "What do critics argue?" These make compelling video segments.
5
Creator uses platform during production
Deliver report + visual screenshots. Platform access for Ask (fact-checking during editing) and Map (generating additional visuals as the script evolves).
Podcast Prep 1–2 days
Content Creator
Walk into interviews knowing the full evidence landscape. Source quotes for reference, What-If for prepping against pushback.
1
Build corpus on the episode topic
Create project. Run Discovery + core pipeline (Gate → Extraction → Results → Meta-Analysis). Speed over depth — Rapid preset works well.
2
Generate talking points via Ask
Open Ask. Ask the questions the audience would ask: "What does the research actually show?", "What's the strongest evidence for X?", "What are the common misconceptions?" Each answer becomes a segment.
3
Prep counter-arguments
Open What If. Test the guest's likely claims: filter to only their methodology — does it hold? Open Ask: "What would critics of X argue?" This prepares the host to push back intelligently.
4
Creator uses Ask tab live during recording
Give platform access. During recording, they can quickly check a claim via Ask: "Is it true that X?" — answer in 5 seconds with citations. Makes for authoritative live responses.
Twitter / LinkedIn Threads 1–2 days
Content Creator
Each key finding is a tweet. Each verified quote is screenshot-worthy. One research brief = 10+ threads.
1
Build corpus and run pipeline
Create project, run Discovery + pipeline through Meta-Analysis. Include Evidence Grounding for verified quotes.
2
Mine thread hooks via Ask
Open Ask. Ask provocative angles: "What's the most surprising finding?", "What does everyone get wrong about X?", "What changed in the last 3 years?" Each answer is a thread hook.
3
Pull stats and quotes
Open Insights for chart-worthy stats. Export verified quotes — these are screenshot-safe for "here's what the paper actually says" posts. The evidence table has numbers for carousel content.
4
Creator uses Ask + Questions for ongoing content
Platform access. Use Ask for new angles as engagement data comes in ("my audience is asking about Y — what does the evidence say?"). Use Questions tab to track which angles have been covered.
Course Material 3–5 days
Content Creator
Build curriculum on structured evidence. Reference library for students, evidence table as teaching resource, requery for each module.
1
Build comprehensive corpus on course topic
Create project with the course's overarching question. Run Discovery in Full mode for comprehensive coverage. Run full pipeline including Credibility for quality tiers.
2
Generate module-specific reports via Questions
Open Questions tab. Submit one question per module/lesson: "What are the foundations of X?", "What methods are used to study Y?", "What are the current debates in Z?" Each generates a focused report.
3
Build reading lists from evidence table
Export CSV and BibTeX. Filter the evidence table by topic/method to create per-module reading lists. T1 papers become required reading, T2 recommended, T3 supplementary.
4
Create visual teaching aids
Open Map — capture evidence network for lecture slides showing how the field connects. Open Insights — use methodology distribution and year trends as teaching context.
5
Creator uses Ask for student questions
Platform access. When students ask unexpected questions, the creator can check Ask in seconds and respond with cited evidence. Builds immense credibility in live sessions.
Book Research 5–7 days
Content Creator
Evidence base per chapter. Ask the Corpus as a writing companion. Every claim traceable to a source before it reaches an editor.
1
Build master corpus for the book's thesis
Create project with the book's central argument as the research question. Run Discovery with broad terms. Run full pipeline with all quality stages — this is a long-term evidence base.
2
Generate chapter-level reports
Open Questions. Submit one question per chapter. Each generates a focused report with chapter-relevant citations and findings. Use different tones: academic for methodology chapters, insight for narrative chapters.
3
Stress-test the book's argument
Open What If. The book's thesis needs to survive scrutiny — filter by methodology, exclude weak studies, test temporal trends. Open Ask: "What is the strongest counter-argument to X?" Address this in the book.
4
Build the bibliography
Export BibTeX for the complete corpus. The evidence table serves as the master fact-checking reference for the editorial process.
5
Creator writes with Ask as a research companion
Platform access throughout the writing process. Ask becomes a live research assistant: "What study showed X?", "How many papers support Y?", "What's the citation for Z?" Every claim is verifiable before submission.
